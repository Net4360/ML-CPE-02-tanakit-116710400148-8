LAB 1: Regression (โมเดลการถดถอยสำหรับทำนายค่าตัวเลขแบบต่อเนื่อง)
1. Simple Linear Regression (โมเดลการถดถอยเชิงเส้นอย่างง่ายด้วยตัวแปรเดียว)
pd.read_csv(...)
ใช้ฟังก์ชันของไลบรารี Pandas เพื่อโหลดและอ่านไฟล์ข้อมูลชุดภาพใบหน้าที่ผ่านการสกัดคุณลักษณะด้วยวิธี PCA (face_dataset_v2.csv) เข้ามาเก็บไว้ในตัวแปร df เพื่อเตรียมนำไปใช้งานต่อ
กำหนดตัวแปรต้นและตัวแปรเป้าหมาย (X_simple และ y)
เลือกระบุคอลัมน์ PCA_Component_1 มาเพียง 1 ตัวแปร เพื่อใช้เป็นตัวแปรอิสระ (Feature) สำหรับการทำนาย และกำหนดให้คอลัมน์ Age เป็นค่าเป้าหมาย (Label/Target) ที่โมเดลต้องทำนาย
train_test_split(...)
เรียกใช้ฟังก์ชันสำหรับแบ่งข้อมูลออกเป็น
ชุดฝึกสอน (Train Set) 80%
ชุดทดสอบ (Test Set) 20%
พร้อมกำหนด random_state = 42 เพื่อให้การสุ่มข้อมูลให้ผลลัพธ์เหมือนเดิมทุกครั้งที่รันโปรแกรม
LinearRegression()
สร้างโมเดลประเภท Linear Regression เพื่อเตรียมใช้สร้างสมการเส้นตรงสำหรับการทำนาย
fit(...)
สั่งให้โมเดลเรียนรู้ (Training) จากข้อมูลชุดฝึกสอน โดยคำนวณความสัมพันธ์ระหว่างตัวแปรต้น X_train_s และตัวแปรเป้าหมาย y_train_s
predict(...)
นำข้อมูลชุดทดสอบ X_test_s ส่งเข้าโมเดลที่ผ่านการฝึกแล้ว เพื่อให้โมเดลทำนายค่าอายุ และเก็บผลลัพธ์ไว้ในตัวแปร y_pred_s
mean_squared_error(...)
คำนวณค่าความผิดพลาดกำลังสองเฉลี่ย (Mean Squared Error : MSE) ระหว่างอายุจริงกับอายุที่โมเดลทำนายได้ โดยค่า MSE ยิ่งเข้าใกล้ 0 มากเท่าใด แสดงว่าโมเดลมีความแม่นยำมากขึ้น
r2_score(...)
คำนวณค่า Coefficient of Determination (R² Score) เพื่อวัดความสามารถของโมเดลในการอธิบายความแปรปรวนของข้อมูล โดยค่า R² ยิ่งเข้าใกล้ 1 มากเท่าใด แสดงว่าโมเดลมีประสิทธิภาพในการทำนายสูง

LAB 2: Classification (โมเดลประเภทการจำแนกหมวดหมู่ข้อมูล)
1. Preparing Classification Data (ขั้นตอนการจัดเตรียมข้อมูลสำหรับการจำแนกกลุ่ม)
LabelEncoder().fit_transform(...)
เรียกใช้ตัวแปลงข้อมูลเชิงคุณภาพ (Label Encoder) เนื่องจากข้อมูลในคอลัมน์ Gender เป็นข้อความ เช่น 'Female' และ 'Male' ซึ่งคอมพิวเตอร์ไม่สามารถนำไปคำนวณได้โดยตรง จึงต้องแปลงให้เป็นรหัสตัวเลข (เช่น 0 และ 1) ก่อนนำไปสร้างโมเดล
กำหนดชุดตัวแปรสำหรับการจำแนก (X_cls และ y_cls)
เลือกใช้ฟีเจอร์ PCA_Component_1 และ PCA_Component_2 เป็นตัวแปรอิสระ (Features) สำหรับใช้แบ่งกลุ่มข้อมูล และกำหนดคอลัมน์ Gender ที่ผ่านการแปลงเป็นตัวเลขแล้วเป็นตัวแปรเป้าหมาย (Target)
2. Logistic Regression (โมเดลการถดถอยโลจิสติกสำหรับการแยกคลาส)
LogisticRegression()
เรียกใช้อัลกอริทึม Logistic Regression ซึ่งเป็นโมเดลพื้นฐานสำหรับการจำแนกข้อมูลออกเป็น 2 กลุ่ม (Binary Classification) เช่น การจำแนกเพศชายและเพศหญิง
predict_proba(...)
นอกจากการทำนายผลลัพธ์แล้ว ฟังก์ชันนี้ยังสามารถคำนวณค่าความน่าจะเป็น (Probability) ของแต่ละคลาสได้ โดยแสดงระดับความมั่นใจของโมเดลว่าข้อมูลนั้นมีโอกาสอยู่ในคลาส 0 หรือ 1 มากน้อยเพียงใด
3. Decision Boundary Visualization (การแสดงเขตแดนการตัดสินใจของโมเดล)
np.meshgrid(...)
สร้างชุดพิกัดจำลองในรูปแบบตาราง (Grid Space) ให้ครอบคลุมช่วงค่าของข้อมูลทั้งสองแกน ได้แก่ PCA_Component_1 และ PCA_Component_2 เพื่อใช้ทดสอบการจำแนกพื้นที่ของโมเดล
plt.contourf(...)
นำผลลัพธ์จากการทำนายของโมเดลมาสร้างกราฟแบบระบายสีพื้นหลัง (Decision Boundary) เพื่อแสดงพื้นที่ที่โมเดลจัดว่าเป็นแต่ละคลาส จากนั้นพล็อตจุดข้อมูลจริงลงบนกราฟ เพื่อเปรียบเทียบว่าการแบ่งพื้นที่ของโมเดลสอดคล้องกับข้อมูลจริงมากน้อยเพียงใด
4. Gender Prediction (การทดสอบทำนายเพศจากข้อมูลชุดใหม่)
pd.DataFrame(...)
สร้างข้อมูลจำลองของใบหน้าใหม่จำนวน 1 ตัวอย่าง โดยกำหนดค่าของ PCA_Component_1 และ PCA_Component_2 เพื่อใช้เป็นข้อมูลสำหรับทดสอบโมเดล
ทดสอบการทำนายผล
นำข้อมูลชุดใหม่เข้าสู่โมเดลด้วยคำสั่ง .predict(...) เพื่อให้โมเดลทำนายว่าเป็นเพศใด พร้อมใช้ .predict_proba(...) เพื่อแสดงค่าความน่าจะเป็นของแต่ละคลาส
inverse_transform(...)
หลังจากโมเดลทำนายผลออกมาเป็นรหัสตัวเลขแล้ว ใช้คำสั่งนี้แปลงผลลัพธ์กลับเป็นข้อความเดิม เช่น Female หรือ Male เพื่อให้สามารถอ่านและแปลผลได้ง่าย พร้อมทั้งแสดงเปอร์เซ็นต์ความน่าจะเป็นของผลการทำนาย
5. Confusion Matrix (ตารางเมทริกซ์ความสับสนและการประเมินผล)
confusion_matrix(...)
คำนวณตาราง Confusion Matrix เพื่อเปรียบเทียบผลการทำนายกับข้อมูลจริง โดยแสดงจำนวนข้อมูลที่โมเดลทำนายถูกต้องและทำนายผิดในแต่ละคลาส เช่น
True Positive (TP)
True Negative (TN)
False Positive (FP)
False Negative (FN)
ซึ่งช่วยให้สามารถวิเคราะห์ข้อผิดพลาดของโมเดลได้อย่างละเอียด
การวัดผลเชิงสถิติ (accuracy_score, precision_score, recall_score และ f1_score)
ใช้ฟังก์ชันสำหรับประเมินประสิทธิภาพของโมเดลในหลายด้าน ได้แก่
Accuracy Score : วัดสัดส่วนของข้อมูลที่โมเดลทำนายได้ถูกต้องทั้งหมด
Precision Score : วัดความแม่นยำของผลการทำนายคลาสที่เป็นบวก
Recall Score : วัดความสามารถของโมเดลในการค้นหาข้อมูลที่เป็นบวกได้ครบถ้วน
F1-Score : เป็นค่าเฉลี่ยแบบสมดุลระหว่าง Precision และ Recall ใช้สำหรับประเมินประสิทธิภาพโดยรวมของโมเดล โดยเฉพาะในกรณีที่ข้อมูลแต่ละคลาสมีจำนวนไม่เท่ากัน
